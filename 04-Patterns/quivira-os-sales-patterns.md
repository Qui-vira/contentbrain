# Sales Content Patterns — Quivira OS

## Pattern 1: Problem-Solution-Proof

1. Name the specific pain (manual charting, guessing content, two full-time jobs)
2. Introduce the system as the fix (not a tool — a system)
3. Show how it works in 4 steps (scan, filter, approve, publish)
4. End with time saved or revenue generated

**Use for:** Launch posts, threads, LinkedIn, long-form video

## Pattern 2: Day-in-the-Life Contrast

1. "Before" — describe the manual grind in detail (hours, frustration, missed trades)
2. "After" — describe the automated version (wake up, approve, done)
3. Bridge — "I built a system that does this"
4. CTA — DM for access

**Use for:** TikTok, Reels, story-style tweets

## Pattern 3: Feature Spotlight (Single Feature Deep Dive)

1. Pick one feature (e.g., competitor scraping, approval flow, confluence filtering)
2. Explain what it does in one sentence
3. Show it in action (screenshot, screen recording, or narrative)
4. Connect to buyer outcome (time saved, money made, stress reduced)

**Use for:** Carousel posts, short videos, individual tweets

## Pattern 4: Tech Stack Reveal

1. List every tool/API in the stack
2. Explain what each one does in one line
3. Show they all connect into one system
4. Position yourself as the builder ("I built this")

**Use for:** Authority content, developer audience, behind-the-scenes

## Pattern 5: Pricing Anchor

1. Ask "how much do you spend on [manual work / multiple tools]?"
2. Show the combined cost is higher than Quivira OS
3. Present pricing tiers
4. CTA with DM keyword

**Use for:** Sales-focused posts, follow-up content after launch
