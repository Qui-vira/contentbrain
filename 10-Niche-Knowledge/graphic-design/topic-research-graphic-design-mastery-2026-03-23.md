## Key Principles (The Design Bible)

### 1. Visual Hierarchy Rules
- **F-pattern for mobile**: Eye scans top-left to right, then down-left to right. Place hero element at 25-35% from top (visual center).
- **One dominant element per slide**: If everything screams, nothing does. Pick ONE thing to be loudest.
- **Size = importance**: Title 4.00em, body title 3.00em, description 1.50em. Headlines should be 2.5-3x body text size.
- **Contrast draws the eye**: Use color contrast (red on dark), size contrast (72pt vs 28pt), and weight contrast (bold vs regular) to create hierarchy.
- **Proximity groups related items**: Things that belong together should be close together. Unrelated items need space between them.
- **Alignment creates order**: Left-align body text always. Center-align only for headlines of 1-2 lines max. Never center-align paragraphs.
- **Repetition builds consistency**: Same fonts, same colors, same spacing across all slides. Brand recognition comes from repetition.

### 2. Typography System
- **Maximum 2 font families per design**: One for headlines (bold/condensed sans-serif), one for body (clean readable sans-serif).
- **Font size hierarchy**: Hero 72-80pt > Headline 52-60pt > Subhead 36-42pt > Body 28-32pt > Small 22-24pt > Tag 20-22pt.
- **Minimum 24px body text** on all slides (anything smaller is unreadable on mobile).
- **Line height**: 1.3x for headlines, 1.5x for body text.
- **Letter spacing**: Use on tags and labels (tracking +50-100), NOT on body text.
- **Font pairing rules**: Pair a geometric sans (Montserrat, Poppins) with a humanist sans (Open Sans, Inter) or pair a display font with a neutral font.
- **Bold condensed for headlines**: They grab attention and take less space. Fonts like Impact, Oswald, Bebas Neue, Anton.
- **Readability test**: View on phone with arm extended. If you squint, font is too small.
- **Maximum 3 font sizes per slide**: More than 3 creates chaos.

### 3. Color Theory for Dark Themes
- **60-30-10 rule**: 60% dominant color (dark background), 30% secondary (panel/card colors), 10% accent (red, gold highlights).
- **Dark background colors**: Not pure black (#000000). Use near-black with slight color: #0A0A0F (blue-black), #1A1A2E (navy-black), #0D1117 (GitHub dark).
- **Accent sparingly**: Red and gold should occupy max 20% of slide area. Overuse kills impact.
- **Contrast ratio**: Minimum 4.5:1 for body text, 3:1 for large text (WCAG AA). White (#FFFFFF) on near-black (#0A0A0F) = 19.4:1 (excellent).
- **Color temperature**: Warm accents (red, gold, orange) pop on cool dark backgrounds (navy, dark blue). This is Quivira's palette.
- **Monochromatic depth**: Use different shades of the same hue for panels, borders, and backgrounds to create depth without adding colors.
- **Never use more than 3 accent colors per slide** (plus neutrals like white/gray).

### 4. Layout & Composition Rules
- **Safe zones**: Top 120px (IG profile overlay), bottom 80px (branding), left/right 80px margins.
- **60px gap from left edge** on content slides.
- **150px gap from top** on square (1080x1080) slides.
- **Portrait format (1080x1350)** gives 25% more space than square. Always use 4:5 for IG carousels.
- **Rule of thirds**: Place key elements at intersection points of a 3x3 grid. Never center everything.
- **Breathing room**: No text touching edges. Minimum 80px margin all sides.
- **Consistent spacing**: 40-60px between major sections, 10-14px between text lines, 20-30px inside cards/panels.
- **White space is a design element**: Empty space directs attention. Don't fill every pixel.

### 5. Carousel-Specific Design Patterns

#### Slide Structure
| Slide | Purpose | Design Treatment |
|-------|---------|-----------------|
| 1 (Cover) | Hook — stop the scroll | Lightest overlay (50-55%), most dramatic background, HERO text, minimal elements |
| 2 | Context — why this matters | Slightly heavier overlay (60%), headline + 2-3 sentences |
| 3-7 | Steps/Value — core content | Heavy overlay (65-70%) for readability, glass panels, bullet points |
| 8 | Proof — result/stat | Gold numbers, glass panel with stat, testimonial |
| 9 (CTA) | Action — what to do | Second lightest overlay (50%), large button, strong glow |

#### Hook Text Rules
- First slide text: fewer than 12 words.
- Create a curiosity gap that requires swiping.
- Use specific, measurable outcomes ("16 Tokens" not "Some Tokens").
- Include time-based urgency when possible.

#### Swipe Cues
- "Swipe >" text in gray at bottom of cover slide.
- Arrow graphics pointing right.
- Progress indicators (slide numbers: 3/8).
- Partially visible off-screen elements that imply continuation.

#### Engagement Maximizers
- 7-10 slides is the sweet spot. 10-slide carousels maximize dwell time.
- Each slide must be self-contained (screenshottable) while maintaining narrative flow.
- Place CTAs in middle AND end for long carousels (10+ slides).
- "Save this post" CTAs drive 2x more saves than "Follow for more."

### 6. Depth & Dimension Techniques
- **Glow effects**: Render colored text on transparent layer, apply Gaussian blur (8-14px radius), composite behind crisp text. Creates luminance.
- **Glass-morphism panels**: Semi-transparent fill (alpha 140-160) + subtle 1px border (alpha 40) + rounded corners (16px radius).
- **Gradient orbs**: Radial gradient circles (soft, feathered edges) placed behind content blocks for ambient color depth.
- **Drop shadows**: Dark rectangle offset 4px behind panels/buttons. Creates floating effect.
- **Vignette**: Darken image edges (0-80px gradient from edges inward) to focus attention on center content.
- **Neon glow technique**: Stack multiple shadows with zero offset but increasing blur radii (3px, 5px, 10px) for cumulative glow intensity.
- **Two-pass rendering**: First pass = glow layer with large stroke + blur. Second pass = crisp text on top. Creates professional luminance.

### 7. Background Treatment
- Every slide gets a unique AI-generated background themed to the content.
- **Overlay darkness**: Cover = 50-55%, Content = 60-65%, Data = 65-70%, CTA = 50%.
- **Vignette on every background**: Darker edges, lighter center.
- **Background should reinforce theme, never compete with text.**
- **"No text, no words, no letters"** in every AI image generation prompt.

### 8. Scroll-Stopping Psychology
- **Faces stop scrolls**: Human faces (especially with eye contact) trigger pattern interruption.
- **High contrast stops scrolls**: Dark background + bright accent = instant pattern break.
- **Numbers in headlines**: "16 Tokens" > "The Tokens." Specific numbers create credibility.
- **Incomplete patterns**: The brain wants to complete what it sees. Partial reveals, cut-off text, split images.
- **Movement suggestion**: Diagonal lines, gradient directions, visual flow all create implied movement.
- **Color isolation**: One bright element on an otherwise muted design draws the eye immediately.

## Common Mistakes to Avoid

1. **Using pure black (#000000) backgrounds** — Use near-black with color undertone (#0A0A0F, #1A1A2E)
2. **Too many fonts** — Max 2 families. 3+ looks amateur.
3. **No hierarchy** — If all text is the same size, nothing stands out. Vary size aggressively.
4. **Ignoring mobile** — 90%+ view on phones. Test at 375px width.
5. **Filling every space** — White space is premium. Let designs breathe.
6. **Center-aligning paragraphs** — Left-align body text always. Center only for short headlines.
7. **Flat designs without depth** — Add at least one depth element per slide (glow, shadow, glass panel, gradient orb).
8. **Inconsistent spacing** — If section gap is 40px on slide 2, it must be 40px on slide 7.
9. **Glow on body text** — Glow on headlines only. Body text must be crisp and clean.
10. **Weak first slide** — Cover slide hook makes or breaks the carousel. Spend 50% of effort here.
11. **No swipe cues** — Viewers won't know to swipe without visual hints.
12. **AI-generated text in backgrounds** — Always include "no text, no words" in prompts.

## Step-by-Step: Design a Premium Carousel (Process)

### Step 1: Content First
- Write all slide text before designing anything
- Define the hook, body structure, and CTA
- Determine slide count (7-10 ideal)

### Step 2: Art Direction
- Choose background theme per slide (cinematic, abstract, themed)
- Define color palette (stick to brand: deep black + red + gold)
- Select fonts (1 headline + 1 body font)

### Step 3: Generate Backgrounds
- Use fal.ai with cinematic prompts
- Include "no text, no words, no letters" always
- Generate at 1080x1350 (4:5)

### Step 4: Composite
- Apply overlay (opacity varies by slide type)
- Add vignette to all backgrounds
- Place gradient orbs for ambient depth
- Add glass panels for content cards
- Render text with proper hierarchy
- Apply glow to headlines only
- Add slide counters (2 through N-1)
- Lock branding to bottom-left

### Step 5: Quality Check
- Readability at phone size (375px)
- One dominant element per slide
- Consistent fonts/colors/spacing
- Glow on headlines, NOT body
- Breathing room (80px min margins)
- No AI text in backgrounds
